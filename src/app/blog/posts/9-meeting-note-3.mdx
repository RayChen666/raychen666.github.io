## Image Agent Design

**Question:** How is the memory module / agent process the image and generate the sumantc layout plan for scene and asset agent to 
understand?

**Answer:** separate the image processing function to another image agent

The current design for thr memory module is like:

<CodeBlock
    marginBottom="16"
    style={{ borderRadius: '20px' }}
    codes={[
      {
        code:
`
 Functionality:
        1. Detect the TARGET room type from the user's natural-language command.
        2. Load a REFERENCE IMAGE for that room type (semantic layout grounding).
        3. Read the CURRENT SCENE JSON (ground-truth object state).
        4. Run a two-stage LLM reasoning call:
            Stage A — semantic layout: 
                extract anchor, zones, cluster relations from the reference image.
            Stage B — delta planning: 
                compare current scene against the target layout → decide what to 
                KEEP / REMOVE / ADD, with quantities and descriptions the Asset 
                Agent can consume directly.
        5. Return a unified memory_context dict that the Orchestrator stores in
        state["memory_context"] before handing off to the Asset Agent and
        Scene Agent.
`,
        language: "JSON",
        label: "JSON"
      }
    ]}
/>

This means, we separate step 2, 3, and 4.A to form an image agent, while the memory module import the processing 
result of the image agent and only do step 1, 4.B, and 5.

## AI Hallucination

One issue Saurabh noticed and discussed we me is the **"LLM Hallucination"**. To understand, check links below:

- **Medium post:** [LLM Hallucinations Explained](https://medium.com/@nirdiamant21/llm-hallucinations-explained-8c76cdd82532)

- **IBM blog:** [What Are AI Hallucinations?](https://www.ibm.com/think/topics/ai-hallucinations)

- **ACM TOIS conference paper:** [A Survey on Hallucination in Large Language Models](https://arxiv.org/abs/2311.05232)

To connect this to our use case and system design, we raised a potential symptom case: when we look at the top-to-bottom view of the 
office layout (see picture below), we, as human, clearly know that the 3 triangle-shapes are windows due to our life experience and 
logical reasoining. However, for AI with hallucination, it is likely that they mark those as doors or some furnitures. (It is impossible 
for an office to have 3 dorrs in one-side wall, and this is intuitive to human).
