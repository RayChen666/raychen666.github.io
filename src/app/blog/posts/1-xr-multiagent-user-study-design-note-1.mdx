## Complex Route Design

Due to the tree-node structure of the multi-agent system, at the tree root there is a conditional trigger determining which route 
the user command should go to, as mentioned in the XR-MultiAgent project description. The "Complex Route" is designed for when the 
user sends out a command that does not have clear spatial relationships involved. For example, "re-organize the whole room layout, 
to change this from a living room to an office."

<figure style={{ maxWidth: '768px', margin: '1.5rem auto' }}>
   <img src="/images/blog/1-xr-multiagent-user-study-design-note-1/living-room-layout.jpg" 
    alt="Control panel layout" 
    width="100%"
    style={{ borderRadius: '20px' }}
  />
</figure>

To make this command work, there should be collaboration among multiple agents (memory, asset, scene, verification, and execution). 
The memory agent, or module, extracts information about what a living room layout looks like, so there is image input. Also, due to 
the vagueness of the user's command, the memory module should figure out what kinds of assets should be involved in the office and 
their quantities based on the layout prediction.

The memory module then passes these two pieces of information in JSON format to the asset agent and scene agent. The asset agent 
first determines the objects involved and their quantities, then appends those from the assets folder to the scene database. The 
scene agent determines the spatial positions of these newly added assets according to the layout prediction, the verification agent 
checks spatial coherence, and finally the execution agent carries it out.

## User Study Design

Targeting IEEE VR or CHI really depends on the innovative aspects of the research. Since CHI is the top-tier conference in the HCI 
field, an extensive user study is expected. On the other hand, IEEE VR reviewers will be more focused on the technical aspects of 
VR and how it helps users interact better. The system design is a big boost, while the user study can be smaller in scale compared to CHI.

- Links to: [IEEE VR Conference](https://ieeevr.org/).

- Links to: [ACM CHI 2026](https://chi2026.acm.org/).

- Links to: [ACM CHI 2025](https://chi2025.acm.org/).

I think there are 2 main innovative aspects of this research. The first is the utilization of the multi-agent structure in the system 
design for processing spatial information. Since it handles user prompts through different routes according to the categorization by the 
language parsing agent, it has a faster response time than a single-route system design of similar complexity. However, since this 
research is not targeting an AI or systems conference, the innovation of the system design should be framed around how this design 
helps users interact better with the interface.

To conduct this user study, we can design a comparative condition that utilizes a single-route execution and chain-based structure 
for the multi-agent system. This user study aims to prove that the parallelized version will be more **efficient** than the non-parallelized 
version in improving the user experience.
